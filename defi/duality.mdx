Duality is a protocol-native DEX integrated directly into Neutron's core infrastructure that merges the strengths of orderbooks and AMMs into a unified, capital-efficient design. Unlike traditional DEXs implemented as smart contracts, Duality is built at the protocol level, giving it significant performance advantages and deeper integration with other Neutron features.

## Trading on Duality

<CardGroup cols={2}>
  <Card title="Limit Orders" icon="list-check" iconType="duotone">
    <p>Place buy and sell orders at specific prices, ensuring optimal execution. Duality supports a number of expressive limit order types:</p>
    <ul className="mb-4 list-disc pl-6">
      <li>Fill-Or-Kill: Only execute if the entire order can be filled</li>
      <li>Immediate-Or-Cancel: Execute as much of the order as possible below the specified limit price</li>
      <li>Good-Til-Canceled: Maker order that sits on orderbook until filled or canceled</li>
      <li>Good-Til-Time: Maker order with an expiration time</li>
      <li>Just-In-Time: Good-Til-Time limit order that expires after exactly 1 block</li>
    </ul>
  </Card>
  
  <Card title="Market Swaps" icon="arrows-rotate" iconType="duotone">
    <p>Execute instant token swaps with minimal slippage, making it easy to trade one token for another in just a few clicks.</p>
    <p>MultiHopSwaps Trade through multiple pairs in a single atomic trade. Multihop swaps use on-chain route finding to find the optimal set of trades and avoid front-running.</p>
  </Card>
</CardGroup>
